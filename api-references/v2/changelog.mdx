## 2026-06-12

### New: SGX Company Screener with feature parity to IDX
- Introduces [`/v2/sgx/companies/`](./singapore/screener/sgx-companies) — the same SQL-like (`?where=`, `?order_by=`) and natural-language (`?q=`) query interface as the IDX screener, scoped to 617 SGX-listed companies.
  - **Natural-language queries** with automatic USD -> SGD conversion (`?q=top 5 SGX banks by market cap`).
  - **~85 SGX fields** exposed across 6 categories. Big-cap and bank-specific fields are explicitly labeled (`Big caps only` / `Banks only`) in the field reference.
  - **Yearly bracket notation** supported (`revenue[2023]`, `total_dividend[2024]`, `total_yield[2023]`). Quarterly notation returns `400 QUARTERLY_NOT_SUPPORTED` — SGX has annual data only.
  - **Breaking change**: the previous `/v2/sgx/companies/?sector=...` endpoint returning a flat `[{symbol, company_name}]` array is **removed**. The new screener returns the paginated `{results, pagination}` envelope. Existing callers must update.

### New IDX endpoints
- [`/v2/suspensions/`](./indonesia/news/suspensions) — Current and historical stock suspensions on IDX.
- [`/v2/company/corporate-actions/{symbol}/`](./indonesia/company/corporate-actions) — Dividends, stock splits, rights issues, and other corporate actions per company.
- [`/v2/company/shareholders-composition/{symbol}/`](./indonesia/company/shareholders-composition) — Top shareholder composition and recent ownership changes.
- **Broker family** (7 endpoints) — Per-broker daily trade summaries, top buyers/sellers by symbol, top accumulations/distributions by broker, foreign vs domestic flow, and a curated broker registry with retail/mixed/institutional cohort classification.
  - [`/v2/brokers/`](./indonesia/brokers/broker-registry) — Curated broker registry.
  - [`/v2/brokers/top/`](./indonesia/brokers/top) — Top brokers by volume.
  - [`/v2/broker-summary/{symbol}/`](./indonesia/brokers/broker-summary-by-symbol) and [`/top/`](./indonesia/brokers/broker-summary-top) — Per-symbol broker activity.
  - [`/v2/broker-activity/{broker_code}/`](./indonesia/brokers/broker-activity-by-code) and [`/top/`](./indonesia/brokers/broker-activity-top) — Per-broker top accumulations/distributions.
  - [`/v2/foreign-flow/{symbol}/`](./indonesia/brokers/foreign-flow-by-symbol) — Foreign vs domestic ownership flow.

### New SGX endpoints
- [`/v2/sgx/news/`](./singapore/news/sgx-news) — SGX news articles, filterable by company/tag.
- [`/v2/sgx/filings/`](./singapore/news/sgx-filings) — SGX insider filings.
- [`/v2/sgx/buybacks/`](./singapore/transaction/share-buybacks) — SGX share buyback activity.
- [`/v2/sgx/short-sell/`](./singapore/transaction/short-sell) — SGX short-sell activity.
- [`/v2/sgx/daily/{symbol}/`](./singapore/transaction/daily) — Daily price and volume history.
- [`/v2/sgx/tags/`](./singapore/helper-list/sgx-tags) — Reference list of SGX news/company tags.

### Platform-wide improvements
- **Credit-cost transparency**: Every v2 endpoint now declares its API-credit cost (1 / 2 / 3 credits) in its description. Natural-language `?q=` queries cost 3 credits where supported.
- **USD to IDR currency handling**: Natural-language queries on the IDX screener now auto-convert dollar amounts to IDR using a live FX feed (cached 24h, falls back gracefully if the feed is unreachable). Also includes USD to SGD for SGX v2.
- **Field rename — `idx_ticker`->  `symbol`**: For consistency across IDX/SGX/KLSE, the `idx_ticker` field is renamed to `symbol` throughout v2 IDX request params and response bodies.

## 2026-05-12

- **v1 fully discontinued.** All `/v1/*` endpoints now return HTTP 410 Gone with `Deprecation`, `Sunset`, and `Link: rel="successor-version"` headers pointing to v2. See the [Migration Guide](/get-started/v2/migration-guide).
- API v2 is now the official and only supported version. v2 ships with a feature-enriched company screener powered by natural-language queries (NLQ) for IDX. SGX NLQ support is rolling out next.

## 2026-03-06

- **Migration Guide Released**: Added comprehensive [migration guide](/get-started/v2/migration-guide) with step-by-step instructions for upgrading from v1 to v2
- **v1 Endpoints Available in v2**: Most legacy v1 endpoints are now accessible in v2 with enhanced functionality:
  - **Simple Migration**: Most endpoints can be migrated by simply changing `/v1/` to `/v2/` in the URL
  - **Enhanced Companies by Index**: Previously limited to single index queries, now supports multiple indices with SQL-style filtering and natural language queries
  - **Improved Top Companies Ranking**: Expanded from 8 predefined classifications to support sorting by any available metric with advanced filtering
  - **Advanced Growth Analytics**: Enhanced growth metrics with flexible filtering and natural language query support
- **Unified News API**: Consolidated news endpoints into a single `/v2/news/` endpoint with extension support for IDX companies and mining sources 
   - [https://api.sectors.app/v2/news/?extension=idx](./indonesia/news/news)
- **Comprehensive Mining API Update**: Standardized all Turso-powered mining endpoints for consistency and robustness
  - **Standardized Pagination**: All list endpoints now use `limit` and `offset` for pagination, returning a consistent `results` and `pagination` object
  - **Slug-based Lookups**: Standardized company and site lookups to use `slug` instead of numeric IDs
- **Enhanced Mining Company Data**:
  - Updated `MiningCompanyListView` with new filters for `commodity_type`, `company_type`, and `has_financials`
  - Expanded `MiningCompanyDetailView` with total mining site counts.
  - Added specialized endpoints for `ownership`, `performance`, and `financials` specifically for mining firms
- **Improved Mining License Tracking**:
  - Updated `MiningLicenseListView` with new filters (`license_type`, `activity`) and GeoJSON support for site locations
  - Corrected `winner` field in `MiningLicenseAuctions` to boolean and expanded documentation to include detailed phase timelines and participant lists
- **Production & Resources**:
  - Restructured `ResourcesAndReserves` into an index-and-detail pattern with updated metric fields (`exploration_target`, `total_inventory`)
  - Standardized `TotalProduction` and `ExportDestinations` with correct units and endpoint paths
- **New APIs**:
  - Added `MiningContractView` documentation for tracking owner-contractor relationships
  - Added `CommodityListView` for discovering available mineral data coverage
- **Site Analysis**: Updated `MiningSiteListView` and added `MiningSiteDetailView` for deep dives into specific mining operations including resource/reserve breakdowns
- **API Parameter & Response Refinements**:
  - Corrected `MiningCompanyPerformanceView` and `MiningCompanyFinancialsView` endpoint paths and standardized on single-year results returning `year`, `available_years`, and `data`
  - Refactored `SalesDestinationView` to use `slug` as a path parameter and return a nested object keyed by country
  - Updated `CommodityPriceView` to support `start_year` and `end_year` parameters with a 3-year maximum range
  - Standardized `GlobalCommodityDataView` to return a list of results and restricted query parameters to `country`, `commodity_type`, and `limit`
- **Free Float Analysis**: Added specialized endpoint to retrieve public float percentage for companies, with support for sector-based filtering
  - [GET /v2/free-float/](./free-float)
- **Postman Collection Update**: Released version 1.3 of the comprehensive Postman collection, including all new mining and analytics endpoints

## 2026-01-09

- Adjusted [all mining related endpoints](./mining-companies) to use `/mining` instead of `/insider`
- Updated Postman collection to match the latest API version
- Multiple refinements to the [companies screener endpoint](./companies), including:
  - Added smart FY handling and enhanced accuracy for `q` query parameter
  - Removed `eps` from the direct fields in the `where` query parameter
  - Added `[fieldname]_q` to the available field in the `where` query parameter for querying quarterly data
  - Renamed several field names in the `where` query parameter
  - Removed `desc` query parameter and changed it to `-` prefix in the `order_by` query parameter
  - Added parentheses forcing for arithmetic expression on `order_by` query parameter to avoid ambiguity with the negative sign
  - Added capability to have multiple-level sorting in `order_by` query parameter
  - Added `include_query_values` query parameter
  - Added `query_values` response
  - Enhanced the `llm_translation` response to no longer having `desc` parameter, but adding `-` prefix in the `order_by` parameter for descending order
