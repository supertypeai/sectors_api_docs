### Header

<ParamField header="Authorization" type="string" required>
  Authorization header that should be filled with your Sectors Financial API
  key.
</ParamField>

### Query Parameters

#### Extension Parameter

The `/v2/news/` endpoint now supports **extension** parameter that determine the news source and available filtering parameters:

<ParamField query="extension" type="string" default="idx">
  - `idx` (default) – Indonesian Stock Exchange news with filters for sectors, subsectors, tags, and tickers
  - `mining` – Mining industry news with keyword search and commodity type filters
</ParamField>

**Note:** The other query parameters are extension-specific. Using IDX parameters with `extension=mining` (or vice versa) will return a 400 error.

#### Shared Parameters between Extensions

<ParamField query="start" type="string">
  ISO date (YYYY‑MM‑DD) for filtering articles on or after this date.
</ParamField>
<ParamField query="end" type="string">
  ISO date (YYYY‑MM‑DD) for filtering articles on or before this date.
</ParamField>
<ParamField query="limit" type="integer" default="20">
  Number of items per page. Maximum is 30.
</ParamField>
<ParamField query="offset" type="integer" default="0">
  Number of items to skip for pagination.
</ParamField>

#### IDX Extension Parameters (`extension=idx`)

<ParamField query="sector" type="string">
  Comma‑separated list of sector slugs to filter by.
  <AccordionGroup>
    <Accordion title="Get the available sector">
      Get the available `sector` from the
      [Subsectors](./helper-list/subsectors) endpoint.
    </Accordion>
    <Accordion title="Examples of the sector">
      `infrastructures`, `transportation-logistic`, `healthcare`
    </Accordion>
    <Accordion title="Format">
      The API endpoint only accepts `sector` in the **kebab case** format (lowercase
      and separated by "-").
    </Accordion>
  </AccordionGroup>
</ParamField>
<ParamField query="sub_sector" type="string">
  Comma‑separated list of subsector slugs to filter by.
  <AccordionGroup>
    <Accordion title="Get the available sub_sector">
      Get the available `sub_sector` from the
      [Subsectors](../helper-list/subsectors) endpoint.
    </Accordion>
    <Accordion title="Examples of the sub_sector">
      `banks`, `financing-service`, `insurance`, `retailing`, `tobacco`
    </Accordion>
    <Accordion title="Format">
      The API endpoint only accepts `sub_sector` in the **kebab case** format
      (lowercase and separated by "-").
    </Accordion>
  </AccordionGroup>
</ParamField>
<ParamField query="tags" type="string">
  Comma‑separated list of tags to filter by.
  <AccordionGroup>
    <Accordion title="Get the available news tags">
      Get the available `tags` from the [News Tags](../helper-list/tags) endpoint.
    </Accordion>
    <Accordion title="Examples of the tags">
      `acquisition`, `allocation-service`, `ai`, `b2b`, `automotive`
    </Accordion>
    <Accordion title="Format">
      The API endpoint only accepts tags in the **kebab case** format
      (lowercase and separated by "-").
    </Accordion>
  </AccordionGroup>
</ParamField>
<ParamField query="tickers" type="string">
  Comma‑separated list of ticker symbols  to filter by (e.g. `BBNI, BBCA, BMRI`).
  <AccordionGroup>
    <Accordion title="Get the available ticker">
      Get the available `ticker` from the
      [Companies Screener](../companies) endpoint.
    </Accordion>
    <Accordion title="Examples of the ticker">
      `ASII`, `BBCA`, `BBRI`, `BMRI`, `BYAN`
    </Accordion>
    <Accordion title="Format">
      A valid `ticker` consists of 4 letters, optionally followed by '.jk', and
      is **case-insensitive**.
    </Accordion>
  </AccordionGroup>
</ParamField>

#### Mining Extension Parameters (`extension=mining`)

<ParamField query="keyword" type="string">
  Full-text search on article titles (case-insensitive, LIKE query).
</ParamField>
<ParamField query="commodity_type" type="string">
  Filter by commodity type (e.g. `coal`, `nickel`). Case-insensitive.
</ParamField>

### Usage Examples

```bash
# IDX news (default extension)
GET /v2/news/

# IDX news with sector filter and limit
GET /v2/news/?extension=idx&sector=Financials&limit=5

# IDX news with ticker filters
GET /v2/news/?extension=idx&tickers=BBCA,BBRI

# Mining news with keyword search
GET /v2/news/?extension=mining&keyword=nickel

# Mining news filtered by commodity and date range
GET /v2/news/?extension=mining&commodity_type=Coal&start=2024-01-01

# Cross-extension error examples (these will return 400 errors)
GET /v2/news/?extension=mining&sector=Finance          # sector is IDX-only
GET /v2/news/?extension=idx&commodity_type=Coal        # commodity_type is mining-only
```

### Response

<ResponseField name="results" type="array">
  Array of news articles. The structure of each article varies by extension:

  <Expandable title="IDX News Article (extension=idx)">
    <ResponseField name="created_at" type="string">
      Creation datetime in ISO 8601 format (UTC).
    </ResponseField>

    <ResponseField name="title" type="string">
      Headline of the news article.
    </ResponseField>

    <ResponseField name="body" type="string">
      Content summary of the article.
    </ResponseField>

    <ResponseField name="source" type="string">
      URL of the news article.
    </ResponseField>

    <ResponseField name="timestamp" type="string">
      Publication datetime in ISO 8601 format.
    </ResponseField>

    <ResponseField name="sector" type="string">
      Sector classification (e.g. `financials`, `technology`).
    </ResponseField>

    <ResponseField name="sub_sector" type="string[]">
      List of subsector tags (e.g. `["banks"]`, `["software-it-services"]`).
    </ResponseField>

    <ResponseField name="tags" type="string[]">
      List of topical tags (e.g. `["Financial Ratios","Bank Indonesia"]`).
    </ResponseField>

    <ResponseField name="tickers" type="string[]">
      List of relevant stock tickers (e.g. `["BBNI.JK", "BBCA.JK"]`).
    </ResponseField>

    <ResponseField name="dimension" type="object | null">
      Scoring dimensions applied to the article (can be `null`):
      <Expandable title="Scoring dimensions">
        <ResponseField name="future" type="integer">Future outlook score.</ResponseField>
        <ResponseField name="dividend" type="integer">Dividend relevance score.</ResponseField>
        <ResponseField name="ownership" type="integer">Ownership impact score.</ResponseField>
        <ResponseField name="technical" type="integer">Technical analysis score.</ResponseField>
        <ResponseField name="valuation" type="integer">Valuation perspective score.</ResponseField>
        <ResponseField name="financials" type="integer">Financial metrics score.</ResponseField>
        <ResponseField name="management" type="integer">Management analysis score.</ResponseField>
        <ResponseField name="sustainability" type="integer">ESG/sustainability score.</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>

  <Expandable title="Mining News Article (extension=mining)">
    <ResponseField name="title" type="string">
      Headline of the news article.
    </ResponseField>

    <ResponseField name="source" type="string">
      URL of the news article.
    </ResponseField>

    <ResponseField name="timestamp" type="string">
      Publication datetime (format: YYYY-MM-DD HH:MM:SS).
    </ResponseField>

    <ResponseField name="commodity_type" type="string[]">
      List of commodity types mentioned in the article (e.g. `["Coal"]`, `["Nickel"]`, `["Gold", "Copper"]`).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pagination" type="object">
  Pagination metadata for the results:
  <Expandable title="Pagination properties">
    <ResponseField name="total_count" type="integer">
      Total number of articles matching the query.
    </ResponseField>

    <ResponseField name="showing" type="integer">
      Number of articles in the current response.
    </ResponseField>

    <ResponseField name="limit" type="integer">
      Maximum number of articles per page (as specified in query).
    </ResponseField>

    <ResponseField name="offset" type="integer">
      Current offset position in the result set.
    </ResponseField>

    <ResponseField name="has_next" type="boolean">
      Whether there are more results available.
    </ResponseField>

    <ResponseField name="has_previous" type="boolean">
      Whether there are previous results available.
    </ResponseField>

    <ResponseField name="next_offset" type="integer | null">
      Offset value for the next page (null if no next page).
    </ResponseField>

    <ResponseField name="previous_offset" type="integer | null">
      Offset value for the previous page (null if no previous page).
    </ResponseField>
  </Expandable>
</ResponseField>
