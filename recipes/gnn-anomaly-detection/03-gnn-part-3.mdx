This is Part 3 of a three-part recipe. Parts 1 and 2 built and trained the GVAE and produced anomaly scores. In this part, we validate those scores against two independent market signals from the Sectors API: broker flow and foreign flow. We then combine all three signals into a single adaptive risk score for each stock.

**Series overview:**

- **Part 1:** Setup, data fetching, feature engineering, graph construction, model definition
- **Part 2:** Training, anomaly scoring, clustering, visualization
- **Part 3 (this article):** Data quality audit, broker flow, foreign flow, combined risk scoring

## Why Confirmation Matters

A GNN reconstruction error is a statistical signal. It tells you that a stock's behavior was inconsistent with what the graph structure would predict. It does not tell you whether that inconsistency reflects a real market event or is a modeling artifact.

Broker flow and foreign flow are independent market signals grounded in actual transaction data. If a stock has a high GNN anomaly score _and_ its top broker is net distributing _and_ foreign investors are net selling, the convergence of three independent signals is much stronger evidence than any one signal alone.

## Cell 12: Broker Flow Analysis

Broker flow data is fetched from the Sectors API v2 endpoint `/v2/broker-summary/{symbol}/top/`. This endpoint returns the top net buyers and top net sellers for a given stock over the analysis period, aggregated across all trading days.

### A Note on the Zero-Sum Trap

The most common mistake when working with this endpoint is to compute a net total by summing all buyer net values and adding them to all seller net values:

```python
# WRONG: always produces a result near zero
total = sum(b["net_idr"] for b in buyers) + sum(s["net_idr"] for s in sellers)
```

Equity markets are zero-sum. For every buyer there is a seller. The top 10 buyers and top 10 sellers in any stock will roughly cancel each other out when summed. The result is always close to zero regardless of the actual market dynamics.

The correct approach is a **dominance score**: compare the single largest accumulator against the single largest distributor.

```python
# CORRECT: dominance score
top_buy_net   = buyers[0].get("net_idr", 0)   # positive: net bought
top_sell_net  = sellers[0].get("net_idr", 0)  # negative: net sold (API returns negative)
net_dominance = top_buy_net + top_sell_net     # positive = accumulation dominant
```

A positive dominance score means the biggest buyer in the market is accumulating more than the biggest seller is distributing. A negative score means distribution is outpacing accumulation.

### Utility Functions

```python
def format_idr(value) -> str:
    """Format IDR values adaptively: Tr / B / M to avoid rounding small values to zero."""
    try:
        if pd.isna(value):
            return "N/A"
    except Exception:
        pass
    sign    = "+" if value >= 0 else "-"
    abs_val = abs(value)
    if abs_val >= 1e12:  return f"Rp {sign}{abs_val/1e12:.2f}Tr"
    elif abs_val >= 1e9: return f"Rp {sign}{abs_val/1e9:.1f}B"
    elif abs_val >= 1e6: return f"Rp {sign}{abs_val/1e6:.0f}M"
    else:                return f"Rp {sign}{abs_val/1e6:.2f}M"


def fetch_broker_top(ticker: str, start: str, end: str,
                     n_brokers: int = 10) -> dict:
    """
    Fetch top buyers and sellers for one stock.

    Endpoint: GET /v2/broker-summary/{symbol}/top/
    No date range limit (unlike /broker-summary/ which caps at 14 days).
    Costs 2 API credits per call.
    """
    url    = f"https://api.sectors.app/v2/broker-summary/{ticker}/top/"
    params = {"start": start, "end": end, "n_brokers": n_brokers}
    resp   = requests.get(url, headers=HEADERS, params=params)
    return resp.json() if resp.status_code == 200 else {}
```

## Cell 13: Foreign Flow Analysis

Foreign flow data is fetched from `/v2/foreign-flow/{symbol}/`. This endpoint returns daily net foreign inflow in IDR for one stock over a date range of up to 90 days.

The sign convention is straightforward: positive `net_foreign_inflow` means foreign brokers were net buyers on that day; negative means net sellers. Because the IDX is a closed market, domestic flow is always the exact negative of foreign flow:

```text
domestic_net = -net_foreign_inflow
```

This means a single endpoint gives you both foreign and domestic sentiment.

```python
def date_chunks(start_str: str, end_str: str, chunk_days: int = 90):
    """Split a date range into chunks of at most chunk_days days."""
    start  = datetime.strptime(start_str, "%Y-%m-%d")
    end    = datetime.strptime(end_str,   "%Y-%m-%d")
    chunks = []
    cur    = start
    while cur < end:
        chunk_end = min(cur + timedelta(days=chunk_days - 1), end)
        chunks.append((cur.strftime("%Y-%m-%d"), chunk_end.strftime("%Y-%m-%d")))
        cur = chunk_end + timedelta(days=1)
    return chunks


def fetch_foreign_flow_v2(ticker: str, start: str, end: str) -> pd.DataFrame:
    """
    Fetch daily net foreign inflow for one stock.

    Endpoint: GET /v2/foreign-flow/{symbol}/
    Max range: 90 days per request. Chunked automatically for longer periods.
    Response field: net_foreign_inflow (positive = foreign net buyers)
    Costs 1 API credit per call.
    """
    chunks   = date_chunks(start, end, chunk_days=90)
    all_rows = []

    for chunk_start, chunk_end in chunks:
        url    = f"https://api.sectors.app/v2/foreign-flow/{ticker}/"
        params = {"start": chunk_start, "end": chunk_end}
        resp   = requests.get(url, headers=HEADERS, params=params)

        if resp.status_code != 200:
            continue

        for point in resp.json().get("data", []):
            all_rows.append({
                "date":            point.get("date", ""),
                # Rename to foreign_net_buy for consistency with downstream code
                "foreign_net_buy": point.get("net_foreign_inflow", 0),
                "ticker":          ticker,
            })

    if not all_rows:
        return pd.DataFrame()

    df           = pd.DataFrame(all_rows)
    df["date"]   = pd.to_datetime(df["date"])
    return df.set_index("date").sort_index()


# Fetch foreign flow for all stocks
```

## Cell 14: Combined Risk Score

The final step merges GNN scores, broker dominance, and foreign flow into a single risk score per stock. Each signal is normalized to a 0 to 1 scale and combined with adaptive weights that depend on which signals are available for each stock.

### Normalization

```python
def min_max_norm_safe(series: pd.Series) -> pd.Series:
    """
    Min-max normalize a series to [0, 1].
    NaN values are excluded from range computation but preserved in output.
    """
    valid = series.dropna()
    if len(valid) < 2 or valid.max() == valid.min():
        return pd.Series(0.5, index=series.index)
    return (series - valid.min()) / (valid.max() - valid.min())
```

```python
    """Format IDR values: Tr / B / M."""
```

## Common Errors and Fixes

### Broker values showing as zero

The zero-sum trap described earlier. Sum of all buyer net values plus sum of all seller net values will cancel. Use dominance score (top buyer vs top distributor) instead.

### NaN in combined\_risk\_score

Usually caused by running `.merge()` on a DataFrame that already has the column from a previous run. pandas does not overwrite existing columns on merge; it creates `column_x` and `column_y` variants. Use `.map()` instead:

```python
# Correct approach: always use .map() not .merge() for signal injection
confirm_df["broker_net_total"] = confirm_df["ticker"].map(
    broker_result_all_df.set_index("ticker")["broker_net_total"]
)
```

### HTTP 400 on broker endpoint

Check that `START_DATE` and `END_DATE` are within the API's supported range. The `/v2/broker-summary/{symbol}/top/` endpoint has no documented date limit, but very long periods (over 365 days) may return errors or empty results depending on your API plan.

### Identical anomaly scores for two tickers

This always indicates zero-feature nodes (both tickers had no data so both were processed as identical zero vectors). Remove them with the Cell 11 audit before running broker and foreign flow.
