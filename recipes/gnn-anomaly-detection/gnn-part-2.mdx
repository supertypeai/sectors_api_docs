## Cell 6: Training Loop

The training loop is straightforward. Each epoch runs a full forward pass, computes the combined loss, backpropagates, clips gradients, and steps the optimizer and scheduler. Progress is printed every 20 epochs.

```python
def train_epoch(model, data, optimizer):
    model.train()
    optimizer.zero_grad()
    x_recon, mu, logstd, z = model(data.x, data.edge_index)
    total_loss, recon_loss  = model.loss(
        data.x, x_recon, mu, logstd, data.edge_index
    )
    total_loss.backward()
    # Clip gradients to prevent instability
    torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
    optimizer.step()
    return total_loss.item(), recon_loss.item()


print(f"Starting training for {EPOCHS} epochs...")
loss_history = []

for epoch in range(1, EPOCHS + 1):
    total_loss, recon_loss = train_epoch(model, graph_data, optimizer)
    scheduler.step()
    loss_history.append({
        "epoch": epoch,
        "total": total_loss,
        "recon": recon_loss
    })

    if epoch % 20 == 0 or epoch == 1:
        print(f"  Epoch {epoch:4d}/{EPOCHS} | "
              f"Total: {total_loss:.4f} | "
              f"Recon: {recon_loss:.4f} | "
              f"LR: {scheduler.get_last_lr()[0]:.5f}")

loss_df = pd.DataFrame(loss_history)
print(f"\nTraining complete. Final total loss: {loss_df['total'].iloc[-1]:.4f}")
```

A healthy training run looks like this: total loss starts around 3.0 and drops rapidly in the first 25 epochs, then continues to decrease more slowly and stabilizes below 0.7 by epoch 150. Reconstruction loss tracks below total loss because it does not include the KL and link terms.

If the loss oscillates without converging, reduce `LEARNING_RATE` from 0.005 to 0.001. If it collapses to near zero within 10 epochs (posterior collapse), increase the KL weight from 0.5 to 1.0 in the loss function.

<Info>
  Gradient clipping at 1.0 is important here. GCN layers can produce large gradient norms when node degrees vary significantly, which is common in a correlation graph where hub stocks like BMRI are connected to many others while smaller stocks may have only a few edges.
</Info>

## Cell 7: Anomaly Scoring

After training, the model is switched to evaluation mode. The full graph is passed through once to get reconstructed features and latent embeddings. Per-node reconstruction error is computed as the mean squared error across all 11 features.

```python
model.eval()
with torch.no_grad():
    x_recon, mu, logstd, z = model(graph_data.x, graph_data.edge_index)

# Reconstruction error per node, per feature dimension
recon_error    = F.mse_loss(x_recon, graph_data.x, reduction="none")

# Average across feature dimensions to get a scalar score per node
anomaly_scores = recon_error.mean(dim=1).cpu().numpy()   # shape: (n_stocks,)
embeddings     = z.cpu().numpy()                          # shape: (n_stocks, 32)

# Threshold at the 90th percentile
threshold  = np.percentile(anomaly_scores, ANOMALY_PERCENTILE)
is_anomaly = anomaly_scores > threshold

# Build result DataFrame
result_df = pd.DataFrame({
    "ticker":        active_tickers,
    "anomaly_score": anomaly_scores,
    "is_anomaly":    is_anomaly,
}, index=range(len(active_tickers)))

# Attach original features for later analysis
result_df = result_df.join(feat_df.reset_index(drop=True))

anomaly_tickers = result_df[result_df["is_anomaly"]]["ticker"].tolist()
normal_tickers  = result_df[~result_df["is_anomaly"]]["ticker"].tolist()

print(f"Anomaly score range: {anomaly_scores.min():.4f} to {anomaly_scores.max():.4f}")
print(f"Threshold (p{ANOMALY_PERCENTILE}): {threshold:.4f}")
print(f"\nAnomalies detected ({len(anomaly_tickers)}): {anomaly_tickers}")
```

```text
Anomaly score range: 0.0176 to 0.9454
Threshold (p90): 0.5319

Anomalies detected (3): ['BBCA', 'PGAS', 'SMGR']
```

<Info>
  The 90th percentile threshold means roughly the top 3 stocks in a universe of 30 will be flagged. Adjust `ANOMALY_PERCENTILE` to change the sensitivity. Lower values (e.g. 85) flag more stocks; higher values (e.g. 95) flag fewer but only the most extreme cases.
</Info>

## Cell 8: Clustering in Embedding Space

Clustering is done in two passes with different goals.

**KMeans on all stocks** maps the overall market structure into four groups. Stocks in the same KMeans cluster share similar graph-contextual representations regardless of whether they are anomalous or not.

**DBSCAN on anomalous stocks only** asks a more specific question: among the stocks that are already flagged as anomalous, do any of them share a common pattern, or did each become anomalous for independent reasons?

```python
# Reduce embedding dimensions for visualization and clustering
pca           = PCA(n_components=2, random_state=42)
embeddings_2d = pca.fit_transform(embeddings)
print(f"PCA variance explained: {pca.explained_variance_ratio_.sum():.2%}")

# KMeans: structure of the full market
kmeans         = KMeans(n_clusters=N_CLUSTERS, random_state=42, n_init=10)
cluster_labels = kmeans.fit_predict(embeddings)

# DBSCAN: structure of anomalies only
anomaly_mask = is_anomaly
if anomaly_mask.sum() >= 3:
    dbscan                = DBSCAN(eps=0.8, min_samples=2, metric="euclidean")
    anomaly_embeddings    = embeddings[anomaly_mask]
    anomaly_cluster_labels = dbscan.fit_predict(anomaly_embeddings)
    n_anomaly_clusters    = len(set(anomaly_cluster_labels)) - (
        1 if -1 in anomaly_cluster_labels else 0
    )
    print(f"DBSCAN: {anomaly_mask.sum()} anomalies -> {n_anomaly_clusters} clusters")
else:
    anomaly_cluster_labels = np.zeros(anomaly_mask.sum(), dtype=int)
    n_anomaly_clusters     = 1

# Attach cluster labels to results
result_df["kmeans_cluster"] = cluster_labels
result_df.loc[result_df["is_anomaly"], "anomaly_cluster"] = (
    anomaly_cluster_labels.astype(int)
)

# Silhouette score for KMeans quality
if len(set(cluster_labels)) > 1:
    sil_score = silhouette_score(embeddings, cluster_labels)
    print(f"KMeans silhouette score: {sil_score:.4f}")

# Summary of anomalies per DBSCAN cluster
anom_df = result_df[result_df["is_anomaly"]][[
    "ticker", "anomaly_score", "anomaly_cluster",
    "n_extreme_days", "n_volume_spikes", "volatility", "skewness"
]].copy()
anom_df = anom_df.sort_values(
    ["anomaly_cluster", "anomaly_score"], ascending=[True, False]
)
print("\nAnomalies by DBSCAN cluster:")
print(anom_df.to_string(index=False))
```

When DBSCAN assigns all anomalies to cluster -1 (the noise cluster), that is not a failure. It means the anomalies are isolated from each other in embedding space and should be investigated separately. When anomalies cluster together, it suggests a shared mechanism worth investigating.

## Reading the Results

### Training Loss Curve (Panel 1)

A loss that drops sharply in the first 25 epochs then plateaus indicates the model has learned the main structure of the data. Continued slow decline after that is fine. A flat curve from epoch 1 indicates the learning rate is too low.

### Anomaly Score Distribution (Panel 2)

The distribution should be right-skewed: most stocks cluster near zero with a long tail of higher scores. The vertical threshold line separates normal from anomalous. If the threshold sits in a gap between two distinct clusters in the distribution, that is a good sign. If it cuts through a dense cluster, consider adjusting `ANOMALY_PERCENTILE`.

### PCA Embedding Space (Panel 3)

Stocks that cluster tightly in this space share similar graph-contextual representations. Anomalies that appear isolated from the main cluster are stocks whose behavior the GNN found hardest to explain given their graph neighbors. Anomalies that appear close to each other despite being flagged independently may share a root cause.

### Correlation Graph (Panel 8)

Node size is proportional to anomaly score. A large node in the center of the graph (a hub with many edges) that is also anomalous is the most interesting case: the GNN had a lot of neighbor information to work with, yet still could not reconstruct the stock's behavior. BMRI appeared this way in the sample run.

### DBSCAN Clusters (Panel 9)

Points labeled as "Outlier" (cluster -1) are anomalies that are isolated from all other anomalies in embedding space. This means they became anomalous for different reasons. In the sample run, all three anomalies were labeled as outliers, suggesting three independent anomaly mechanisms rather than a single shared cause.
